  <behavior>
    Cases:
    - Happy path: scout_acquire("weather tools", ctx) calls search_registry, filter_http_servers, rank_servers, proxy.connect with top result, fires ToolListChangedNotification, returns {"status": "connected", "server": name, "url": url, "score": int, "reasoning": str}
    - Cache hit: When proxy.connect returns "already_connected", scout_acquire returns {"status": "already_connected", ...} and does NOT fire ToolListChangedNotification
    - No HTTP servers: When filter_http_servers returns [], scout_acquire returns {"status": "no_servers_found", "context": context} without calling rank_servers
    - No ranked results: When rank_servers returns [], scout_acquire returns {"status": "no_servers_found", "context": context}
    - Signature check: scout_acquire has parameters (context: str, ctx: Context) and is async
  </behavior>
  <implementation>
    Add scout_acquire to server.py following the exact pattern from 03-RESEARCH.md:

    ```python
    @mcp.tool
    async def scout_acquire(context: str, ctx: Context) -> dict:
        """Find the best MCP server for a task and connect to it immediately."""
        raw = await search_registry(context)
        http_servers = filter_http_servers(raw)
        if not http_servers:
            return {"status": "no_servers_found", "context": context}
        ranked = await rank_servers(context, http_servers)
        if not ranked:
            return {"status": "no_servers_found", "context": context}
        top = ranked[0]
        result = proxy.connect(top["name"], top["remote_url"], mcp)
        if result["status"] == "connected":
            await ctx.send_notification(mcp_types.ToolListChangedNotification())
        return {
            "status": result["status"],
            "server": top["name"],
            "url": top["remote_url"],
            "score": top.get("score"),
            "reasoning": top.get("reasoning"),
        }
    ```

    Place above `if __name__ == "__main__":` block. Keep all existing imports (mcp_types alias, proxy, ranker imports are already present).

    CRITICAL anti-patterns to avoid:
    - Do NOT fire notification on "already_connected" status
    - Do NOT access ranked[0] without guarding for empty list
    - Do NOT use `mcp.types` directly -- use `mcp_types` alias (module-level mcp = FastMCP shadows the package)
  </implementation>
</feature>
