<feature>
  <name>scout_list_active and scout_disconnect</name>
  <files>server.py, tests/test_scout_control.py</files>
  <behavior>
    scout_list_active:
    - Returns [{"name": "srv1", "url": "https://..."}, ...] when servers are connected
    - Returns [] when no servers are connected
    - Is synchronous (def, not async def) since proxy.get_connections() is sync
    - Has no parameters

    scout_disconnect:
    - Calls proxy.disconnect(server_name, mcp) and returns the result
    - Fires ToolListChangedNotification when result status is "disconnected"
    - Does NOT fire notification when result status is "not_found"
    - Has parameters (server_name: str, ctx: Context) and is async
  </behavior>
  <implementation>
    scout_list_active (sync, no ctx needed):
    ```python
    @mcp.tool
    def scout_list_active() -> list[dict]:
        """List all MCP servers Scout has connected to in this session."""
        return [
            {"name": name, "url": url}
            for name, url in proxy.get_connections().items()
        ]
    ```

    scout_disconnect (async, needs ctx for notification):
    ```python
    @mcp.tool
    async def scout_disconnect(server_name: str, ctx: Context) -> dict:
        """Disconnect a server and remove its tools from the host's tool list."""
        result = proxy.disconnect(server_name, mcp)
        if result["status"] == "disconnected":
            await ctx.send_notification(mcp_types.ToolListChangedNotification())
        return result
    ```

    Place both AFTER scout_acquire (or scout_connect if Plan 01 hasn't run yet) and BEFORE `if __name__ == "__main__":`.

    CRITICAL anti-patterns to avoid:
    - Do NOT make scout_list_active async -- no await is needed
    - Do NOT cache the result of get_connections() at import time -- call it fresh each invocation
    - Do NOT fire notification on "not_found" status in scout_disconnect
  </implementation>
</feature>
